# 技術規格文件
## 明洪武年間藩國朝貢互動地圖 — 開發指南

---

## 一、前序：技術方案審查與修正

在制定方案前，需先指出原始構想中的三個核心技術風險及應對策略。

### 風險1：古地圖PNG坐標映射不可靠

**問題**：歷史古地圖不使用任何標準地圖投影。如果直接將lat/lng線性映射到PNG像素，藩國位置將嚴重偏移。

**解決方案 — 雙層底圖架構**：
- **幾何層（Canvas自繪）**：用簡化的現代海岸線GeoJSON，以Mercator投影渲染到Canvas。這一層是坐標計算的唯一參考，保證藩國位置精確。風格化處理為古地圖質感（粗筆觸、紙紋色）。
- **裝飾層（PNG）**：古地圖PNG以低透明度（opacity 0.15–0.25）疊加在幾何層之下，作為視覺紋理和歷史氛圍裝飾。它不參與任何坐標計算。
- **如果找不到合適的古地圖PNG**：直接用Canvas幾何層 + CSS紙紋背景，效果也足夠古樸。不要因為底圖PNG阻塞開發。

### 風險2：Canvas無原生DOM事件（點擊檢測）

**問題**：Canvas繪製的圓點不是DOM元素，無法直接綁定click事件。手寫hit-testing代碼容易出bug。

**解決方案 — HTML熱區疊加層**：在Canvas上方放一層HTML `<div>`，為每個藩國建立一個絕對定位的透明小方塊（30×30px），作為點擊和hover的目標。HTML元素的位置由同一個 `latLngToPixel()` 函數計算，與Canvas繪製完全同步。這樣點擊事件、hover效果、cursor:pointer全部用原生HTML/CSS處理，零額外代碼。

### 風險3：朝貢路線不應是直線弧

**問題**：東南亞到南京的實際朝貢路線是海路（沿中南半島東岸北上，經廣東、福建沿海至長江口，入南京）。高麗/朝鮮則走陸路經遼東。直線弧或簡單拋物線會顯得不合理。

**解決方案 — 分類路線模板**：
| 路線類型 | 適用藩國 | 控制點策略 |
|---------|---------|-----------|
| 東南亞海路 | 占城、安南、暹羅、真臘、爪哇、三佛齊、浡泥等 | 沿南海北上 → 福建沿海 → 長江口 → 南京 |
| 東海路線 | 日本、琉球三國 | 東海直渡 → 寧波/福建 → 沿海北上 → 南京 |
| 陸路/混合 | 高麗/朝鮮 | 渡鴨綠江 → 遼東 → 山東 → 南京 |
| 西域路線 | 瑣里、西洋、拂菻、鬚文那達 | 印度洋 → 麻六甲 → 南海 → 廣東 → 南京 |

每條路線預定義3–5個路徑控制點，用三次Bezier曲線平滑連接。路線數據硬編碼在配置文件中。

## 二、整體技術架構

### 2.1 技術棧

```
前端唯一：HTML5 + CSS3 + Vanilla JavaScript（ES6+）
無構建工具：直接打開 index.html 即可運行
無後端依賴：所有數據嵌入JS文件
外部CDN：無（完全離線可用，便於評審直接打開）
```

**選擇理由**：比賽評審場景需要零配置啟動。React/Vue需要npm build；D3需要CDN。純Vanilla JS確保雙擊HTML文件即可看到完整效果。

### 2.2 渲染層架構

從下到上共6層，使用CSS `position: absolute` 堆疊在同一個容器內：

```
┌─────────────────────────────────────┐
│  z-6  #ui-layer     時間軸面板、標題  │  HTML div
│  z-5  #label-layer   藩國名稱標籤     │  HTML div（每個標籤一個子div）
│  z-4  #hit-layer     點擊熱區        │  HTML div（透明方塊）
│  z-3  #anim-canvas   車馬動畫        │  Canvas（pointer-events:none）
│  z-2  #route-canvas  軌跡線+圓點     │  Canvas（pointer-events:none）
│  z-1  #base-canvas   海岸線+地名     │  Canvas（隻畫一次）
│  z-0  CSS background 紙紋/古地圖     │  CSS background-image
└─────────────────────────────────────┘
```

**性能特性**：
- `#base-canvas`：頁面加載時繪製一次，之後永不重繪（緩存為離屏Canvas）
- `#route-canvas`：僅在滑桿值改變時重繪（每次滑桿input事件觸發）
- `#anim-canvas`：僅在動畫播放時使用requestAnimationFrame，動畫結束後停止循環
- HTML層：DOM操作隻在滑桿值改變時批量更新

```

│   ├── baseMap.js          海岸線繪製（#base-canvas）
```

### 2.4 核心數據模型

#### 朝貢記錄（TributeRecord）
```javascript
{
  id: "champa_02_02",           // countryId_年_月
  countryId: "champa",          // 藩國英文ID
  countryName: "占城",          // 藩國中文名
  year: 2,                      // 洪武年份（整數2–31）
  month: 2,                     // 月份（整數1–12）
  isLeapMonth: false,           // 是否閏月
  gregorianYear: 1369,          // 對應公元年（year + 1367）
  king: "阿答阿者",             // 國王（可空字串）
  envoy: "虎都蠻",              // 使者（可空字串）
  purpose: "",                   // 目的（可空字串）
  tributeRaw: "虎、象、方物",    // 原始貢物文本
  tributeItems: ["虎","象","方物"],  // 分詞後貢物列表
  tributeCategories: ["animal","misc"],  // 對應類別
  giftsRaw: "大統歷一本...",     // 原始給賜文本
  source: "《明太祖實錄》卷39"   // 史料來源（保留原文）
}
```

#### 藩國配置（CountryConfig）
```javascript
{
  id: "champa",
  name: "占城",
  nameEn: "Champa",
  lat: 15.5,                    // 緯度（核心都城位置）
  lng: 108.9,                   // 經度
  region: "southeast_asia",     // 區域分類（決定路線模板）
  routeType: "sea_south",       // 路線類型（見2.3節路線分類）
  totalRecords: 27,             // 總記錄數（供子頁面統計用）
  note: ""                      // 特殊說明（如政權更替）
}
```

#### 路線模板（RouteTemplate）
```javascript
{
  type: "sea_south",
  waypoints: [                  // 從藩國到南京的中間控制點
    { lat: 20.0, lng: 110.0 },  // 海南島東側
    { lat: 25.0, lng: 118.0 },  // 福建沿海
    { lat: 30.0, lng: 121.0 },  // 長江口
  ]
  // 起點 = 藩國坐標，終點 = 南京 (32.06, 118.78)
  // 控制點與起終點組合後用Catmull-Rom或三次Bezier平滑
}
```

### Phase 0：數據工程

**目標**：將原始DOCX表格轉換為結構化JavaScript數據文件，並建立全部輔助配置。

**前置條件**：原始DOCX文件已提取文本。

**交付物**：`data.js`、`config.js`

#### 任務清單

1. **解析原始文本**
   - 按行拆分，識別表格結構（| 分隔）
   - 提取8個字段：時間、藩國、國王、使者、目的、貢物、給賜、來源
   - 處理加粗標記（`**`）的清除

2. **時間字段解析**
   - 中文數字轉阿拉伯數字（二→2，二十六→26）
   - 識別閏月標記（「閏」前綴），設 `isLeapMonth: true`
   - 計算公元年：`gregorianYear = year + 1367`

3. **藩國名稱規範化**
   - `高麗（朝鮮）` → 視上下文歸入 `goryeo` 或 `joseon`
   - `琉球`（泛稱）→ `ryukyu_chuzan`
   - `爪哇東番` / `爪哇西番` → 保留獨立ID但共用爪哇坐標
   - 建立ID映射表：中文名 → countryId

4. **貢物分詞與分類**
   - 按頓號（`、`）和逗號分割貢物字段
   - 去除數量詞（「五十一隻」→「象」；「七萬五千斤」→「胡椒」）
   - 匹配到5個類別：animal / spice / textile / vessel / misc
   - 未匹配項歸入misc

5. **藩國坐標表**
   - 20個政治實體的lat/lng（以其都城或核心區域為準）
   - 坐標須落在地圖覆蓋範圍內（lat 0°–50°N, lng 90°–150°E）

6. **路線控制點配置**
   - 4類路線模板各含3–5個waypoint
   - 每個藩國指定所屬路線類型

### Phase 1：靜態底圖渲染

**目標**：在瀏覽器中顯示一幅完整的東亞/東南亞底圖，繪製海岸線並標註南京和全部20個藩國位置。

**前置條件**：Phase 0已通過全部檢查點。

**交付物**：`index.html`、`css/style.css`、`js/baseMap.js`、`js/config.js`（投影參數部分）

#### 任務清單

1. **HTML骨架**
   - 全屏佈局（`100vw × 100vh`，禁止滾動）
   - 地圖容器 `#map-container` 佔滿視口
   - 6層渲染結構（見2.2節），此階段隻實現 `#base-canvas` + `#label-layer` + `#ui-layer`
   - 頂部標題「明洪武年間藩國朝貢圖」

2. **CSS全局樣式**
   - 背景色 `#F5F0E8`（米黃紙紋）
   - 字體 `'Noto Serif TC', 'Noto Serif SC', 'Source Han Serif TC', serif`
   - 所有顏色常量用CSS Custom Properties定義

3. **坐標投影系統**
   - 函數 `latLngToPixel(lat, lng) → {x, y}`
   - 使用等距圓柱投影（Equirectangular）：最簡單，足夠此緯度範圍使用
   - 地圖邊界：`lng [95, 148]`, `lat [0, 50]`
   - 線性映射到Canvas像素：`x = (lng - 95) / (148 - 95) * canvasWidth`, `y = (50 - lat) / (50 - 0) * canvasHeight`
   - Canvas尺寸跟隨容器，支持window.resize重繪

4. **海岸線繪製**
   - 簡化的東亞海岸線路徑（約50–80個控制點，手動編碼）
   - 覆蓋範圍：朝鮮半島 → 中國東海岸 → 中南半島 → 馬來半島 → 印尼群島 → 日本列島
   - 繪製風格：陸地填充 `#E8E0D0`（比背景略深的米色），海岸線描邊 `#8B8070`（灰褐），線寬1.5px
   - 海域無填充（透明，顯示CSS背景色）

5. **地標繪製**
   - 南京（應天府）：特殊標記（城門圖標或大號方形 + 「應天府」標籤），朱紅色
   - 所有藩國：灰色小圓點（r=4px），國名標籤在旁邊（12px，半透明灰色）
   - 標籤避免重疊：琉球三國需垂直錯開；東南亞密集區需調整偏移

### Phase 2：時間軸交互與點亮邏輯

**目標**：實現底部時間滑桿，拖動時對應年份的藩國在地圖上點亮，其餘回到灰色靜默狀態。

**前置條件**：Phase 1已通過全部檢查點。

**交付物**：`js/bindEvents.js`、`js/routeRenderer.js`（圓點部分）、`js/hitLayer.js`、更新 `index.html`

#### 任務清單

1. **時間軸UI**
   - 底部固定面板（高60px），半透明米黃背景
   - `<input type="range" min="2" max="31" step="1">` 滑桿
   - 滑桿左側顯示：「洪武X年」（繁體）
   - 滑桿右側顯示：對應公元年份「（13XX年）」
   - 滑桿下方顯示該年朝貢國家數量：「N國遣使」

2. **數據篩選函數**
   - `getActiveCountries(year) → Set<countryId>`
   - 從 `TRIBUTE_DATA` 中篩選 `year === currentYear`，提取唯一countryId集合
   - 返回值用於控制繪製和HTML更新

3. **圓點繪製（route-canvas）**
   - 激活狀態：朱紅色填充（`#C0392B`），半徑8px，外圈半透明光暈（r=16px, opacity 0.2）
   - 靜默狀態：灰色描邊（`#B0A898`），半徑4px，無填充
   - 每次滑桿input事件：清空route-canvas → 重繪全部圓點

4. **HTML熱區與標籤**
   - 每個藩國一個 `<div class="hit-target">` 絕對定位在Canvas上方
   - 激活時：標籤文字變為墨黑色、加粗；附帶「N次」朝貢次數小標
   - 靜默時：標籤保持半透明灰色
   - 熱區綁定click事件：`window.location.href = 'country.html?id=' + countryId`

5. **3秒靜默計時器初始化**
   - 聲明 `let silenceTimer = null`
   - 每次滑桿input事件：`clearTimeout(silenceTimer)`
   - 此階段不實現動畫回調（Phase 4完成）

### Phase 3：軌跡線與貢物圖標

**目標**：激活的藩國與南京之間繪製朝貢路線弧線，路線上顯示該國該年主要貢物的水墨小圖標。

**前置條件**：Phase 2已通過全部檢查點。

**交付物**：`js/routeRenderer.js`（路線部分）、`js/icons.js`

#### 任務清單

1. **路線繪製算法**
   - 讀取藩國的 `routeType`，取對應路線模板的waypoints
   - 在起點（藩國坐標）、所有waypoints、終點（南京）之間，用Catmull-Rom樣條或串聯的三次Bezier曲線生成平滑路徑
   - 將lat/lng路徑通過 `latLngToPixel` 轉為Canvas像素路徑
   - 繪製到 `#route-canvas`，同一Canvas已有圓點繪製

2. **路線視覺風格**
   - 線型：虛線 `setLineDash([6, 4])`
   - 顏色：金褐 `#8B6914`，透明度0.5
   - 線寬：1.5px
   - 終點（南京端）：小三角箭頭指示方向
   - 每條路線從藩國端漸顯到南京端（可選：沿路徑的透明度從0.3漸變到0.7）

3. **路線路徑緩存**
   - 每條路線的Canvas像素路徑計算一次，緩存為 `Float32Array`
   - 滑桿切換時隻決定顯示/隱藏，不重新計算路徑（除非resize）
   - 路徑點密度：每條路線100個採樣點（用參數t=0到t=1均勻採樣），存為 `{points: [{x,y}, ...], angles: [rad, ...]}`
   - 這個緩存在Phase 4動畫中也會複用

4. **貢物圖標系統（icons.js）**
   - 5個類別各一個Canvas繪製函數，風格為水墨簡筆畫
   - **animal（動物）**：大象剪影 — 圓形身體 + 長鼻 + 四腿，2–3筆勾勒
   - **spice（香料）**：香爐 — 碗狀容器 + 三條上升曲線（香氣），2–3筆
   - **textile（織物）**：卷軸 — 兩端圓柱 + 中間矩形，2筆
   - **vessel（器皿）**：寶瓶 — 花瓶輪廓，2筆
   - **misc（方物）**：箱子 — 正面矩形 + 頂面平行四邊形，2筆
   - 所有圖標墨黑描邊（`#1A1008`），1.5px線寬，無填充或淡灰填充
   - 每個圖標在24×24px bounding box內

5. **圖標放置邏輯**
   - 取該年該國所有記錄的貢物，統計出現的類別
   - 取出現頻率最高的1–2個類別
   - 在路線的t=0.35和t=0.55位置各放一個圖標（避免堆在路線中點）
   - 如果隻有一個類別，隻在t=0.45放一個

### Phase 4：車馬動畫系統

**目標**：用戶在某年份靜默3秒後，使團圖標沿路線向南京行進；滑桿操作立即中斷動畫。

**前置條件**：Phase 3已通過全部檢查點。

**交付物**：`js/caravanAnim.js`，更新 `js/bindEvents.js`

#### 任務清單

1. **使團圖標繪製**
   - Canvas繪製函數 `drawCaravan(ctx, x, y, angle)`
   - 圖案：一面旗幟（朱紅三角形 + 旗桿） + 一個簡化馬匹剪影（橢圓身+四線腿+小圓頭）
   - 整體約32×20px
   - 根據angle旋轉整個圖案（使其朝行進方向）

2. **單路線動畫函數**
   - `startCaravanAnimation(countryId) → cancelFn`
   - 從Phase 3緩存的路徑點數組讀取路線
   - 動畫時長：4000ms（固定）
   - 每幀：計算t值（0→1），從路徑點數組插值當前位置(x,y)和角度
   - 使用requestAnimationFrame驅動
   - 返回一個取消函數（調用時停止該動畫）

3. **#anim-canvas繪製策略**
   - 每幀隻清除 #anim-canvas（透明），不影響 #base-canvas 和 #route-canvas
   - 繪製所有正在進行的使團圖標
   - 動畫結束後清除 #anim-canvas 並停止 requestAnimationFrame

4. **3秒靜默判據完整實現**
   ```
   滑桿input事件觸發 →
     clearTimeout(silenceTimer)
     取消所有正在播放的動畫 → stopAllAnimations()
     清空 #anim-canvas
     更新路線和圓點（Phase 2/3邏輯）
     silenceTimer = setTimeout(() → {
       const activeIds = getActiveCountries(currentYear)
       activeIds.forEach(id → {
         const cancel = startCaravanAnimation(id)
         activeAnimations.set(id, cancel)
       })
     }, 3000)
   ```

5. **多路線並發處理**
   - 用 `Map<countryId, cancelFn>` 管理所有活躍動畫
   - `stopAllAnimations()`: 遍歷Map調用每個cancelFn，然後clear
   - 同年多個國家的使團各自獨立動畫，視覺上同時出發
   - 為避免重疊：如果某年激活國>5個，按路線長度排序，每隔300ms依次啟動（錯開出發）

## 六、已知風險清單

| # | 風險 | 影響 | 緩解方案 |
|---|------|------|---------|
| R1 | 古地圖PNG版權不明 | 作品合法性 | 使用Wikimedia Commons CC授權圖片，或完全依賴Canvas自繪底圖 |
| R2 | 海岸線手繪精度不足 | 視覺效果差 | 從Natural Earth公開GeoJSON提取簡化路徑，轉為Canvas繪製指令 |
| R3 | 藩國坐標學術爭議 | 學術嚴謹度受質疑 | 在作品中註明「位置為近似標註」，附參考文獻 |
| R4 | 290條數據手動解析出錯 | 數據準確性 | Phase 0設立逐年交叉驗證檢查點 |
| R5 | 水墨圖標辨識度不足 | 用戶體驗差 | 圖標旁加文字標籤作為fallback |
| R6 | 快速拖動滑桿導致動畫堆積 | 性能崩潰 | 3秒debounce + stopAllAnimations即時清理 |
| R7 | 拂菻/西洋超出地圖範圍 | 佈局破壞 | 採用邊界標記方式，不擴大地圖範圍 |
| R8 | 評審無法打開HTML | 無法展示 | 提供在線部署版本（GitHub Pages）作為備選 |

## 七、開發次序與依賴關係

```
Phase 0 (數據)
   ↓
Phase 1 (底圖) ← 依賴 Phase 0 的坐標數據
   ↓
Phase 2 (時間軸) ← 依賴 Phase 1 的渲染框架
   ↓
Phase 3 (路線+圖標) ← 依賴 Phase 2 的篩選邏輯
   ↓
Phase 4 (動畫) ← 依賴 Phase 3 的路徑緩存
   ↓
Phase 5 (子頁面) ← 依賴 Phase 0 的數據（可與Phase 3/4並行開發）
   ↓
Phase 6 (打磨) ← 依賴全部Phase完成
```

**並行可能性**：Phase 5（子頁面）隻依賴data.js，理論上可在Phase 2完成後即開始開發，與Phase 3/4並行。
